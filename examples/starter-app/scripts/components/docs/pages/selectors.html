Selectors & DOM — $() & $.all()

zQuery provides two selector functions that mirror the browser's querySelector and querySelectorAll, with added convenience for DOM elements, NodeLists, and HTML string creation.

FunctionReturnsDescription
$(selector)Element | nullSingle element via querySelector
$.all(selector)ZQueryCollectionAll matching elements via querySelectorAll, wrapped in a chainable collection

Both accept CSS selectors, DOM elements, NodeLists, arrays, and HTML strings. $() additionally accepts a function as a DOM-ready shorthand.

Single-Element Selector — $()

// CSS selector — returns first matching element (or null)
$('.card')                     // first .card element
$('li', '#myList')             // first li scoped within #myList

// DOM element — returned as-is
$(document.getElementById('app'))

// ZQueryCollection — returns the first element from the collection
$($.all('.card'))

// NodeList / Array — returns the first element
$(document.querySelectorAll('p'))

// Create element from HTML string — returns first created element
$('<div class="alert">Hello</div>')

// DOM-ready callback (like jQuery $(fn))
$(() => console.log('DOM is ready'))

// Works with standard DOM API — no wrapper overhead
const el = $('.card');
if (el) el.classList.add('active');
When to use $() vs $.all(): Use $() when you expect a single element (a container, a form, a specific ID). Use $.all() when you need to operate on multiple elements (all cards, all list items, all buttons of a class). $() returns a raw DOM element, so you can use native DOM methods directly without any abstraction layer.

Collection Selector — $.all()

// CSS selector — returns ZQueryCollection of all matches
$.all('.card')                    // all .card elements
$.all('li', '#myList')            // all li scoped within #myList

// Wrap a NodeList or array of elements
$.all(document.querySelectorAll('p'))

// Wrap a single DOM element into a collection
$.all(document.getElementById('app'))

// Chainable collection methods
$.all('.card').addClass('active').css({ opacity: '1' });

// Iterable — supports for...of and spread
for (const el of $.all('.item')) {
  console.log(el.textContent);
}
const arr = [...$.all('.item')];  // spread to plain array

Element Creation — $.create()

Create a DOM element programmatically with attributes, styles, event listeners, and children in a single call:

const btn = $.create('button', {
  class: 'primary',
  style: { padding: '10px', borderRadius: '6px' },
  onclick: () => alert('Clicked!'),
  data: { action: 'submit', id: '42' }  // sets data-action, data-id
}, 'Click Me');

document.body.appendChild(btn);
Attribute KeyBehavior
classSets className directly
style (object)Assigns each property to el.style
on* (e.g. onclick)Adds event listener for the event name after "on"
data (object)Sets dataset properties
Anything elseCalls setAttribute(key, value)

Additional arguments after attrs are appended as children (strings become text nodes, elements are appended directly).

Events — Direct & Delegated

// Direct event on all matched elements
$.all('.btn').on('click', (e) => { /* ... */ });

// Multiple events at once (space-separated)
$.all('.btn').on('mouseenter mouseleave', (e) => {
  e.target.classList.toggle('hovered');
});

// Delegated event — listen on parent, filter by child selector
// (This is like jQuery's $(parent).on('click', '.child', fn))
$.all('#list').on('click', '.item', function(e) {
  console.log('Clicked:', this.textContent);
});

// One-time event — auto-removes after first invocation
$.all('.btn').one('click', () => console.log('once'));

// Custom events with detail payload
$.all('.widget').trigger('update', { value: 42 });
$.all('.widget').on('update', (e) => console.log(e.detail));
// e.detail = { value: 42 }
Delegated events are essential for dynamic content. If you add elements to the DOM after binding, delegated events on a parent still work because the check happens at event time, not bind time.

Animations

All animation methods return Promises, so you can await them for sequencing:

MethodSignatureReturnsDescription
animateanimate(props, duration = 300, easing = 'ease')Promise<ZQueryCollection>CSS transition to target properties. Includes a fallback timeout at duration + 50ms.
fadeInfadeIn(duration = 300)PromiseFade in from opacity: 0 to opacity: 1
fadeOutfadeOut(duration = 300)PromiseFade out and hide element after completion
slideToggleslideToggle(duration = 300)thisToggle height between 0 and natural height
// Sequential animation
await $.all('.card').animate({ opacity: '0', transform: 'scale(0.9)' }, 300);
await $.all('.card').fadeIn(400);

// Custom easing
await $.all('.modal').animate(
  { transform: 'translateY(0)' }, 500, 'cubic-bezier(0.16, 1, 0.3, 1)'
);

Form Helpers

// URL-encoded string (like traditional form submission)
$.all('#form').serialize();        // "name=Tony&email=tony%40x.com"

// Plain object (handles duplicate field names as arrays)
$.all('#form').serializeObject();  // { name: 'Tony', email: 'tony@x.com' }
// If multiple checkboxes: { colors: ['red', 'blue'] }

Global Event Delegation — $.on()

Delegate events on the entire document — useful for handling clicks on elements that don't exist yet:

$.on('click', '.delete-btn', function(e) {
  this.closest('.row').remove();
});

Extend with Plugins — $.fn

$.fn is an alias for ZQueryCollection.prototype. Add custom methods to it just like jQuery plugins:

// Define a plugin
$.fn.highlight = function(color = 'yellow') {
  return this.css({ background: color });
};

// Use it
$.all('.important').highlight('#ff0');
$.all('p').highlight();  // defaults to yellow
Tip: Always return this from plugin methods to preserve chainability.
