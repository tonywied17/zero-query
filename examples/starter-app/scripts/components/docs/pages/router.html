Router

A full-featured client-side SPA router with history and hash mode, dynamic route params, query strings, navigation guards, lazy loading, and lifecycle hooks. Handles [z-link] click interception, popstate/hashchange events, and component mounting automatically.

Setup

const router = $.router({
  el: '#app',               // outlet element (required)
  mode: 'history',          // 'history' (default) or 'hash'
  base: '/my-app',          // optional — auto-detected from 
  routes: [
    { path: '/',           component: 'home-page' },
    { path: '/user/:id',   component: 'user-page' },
    { path: '/settings',   component: 'settings-page' },
    { path: '/admin/*',    component: 'admin-page' },  // wildcard
  ],
  fallback: 'not-found'    // 404 component (when no route matches)
});

Config Options

OptionTypeDefaultDescription
elstring | Element—Required. The DOM element (or selector) where matched components are mounted.
mode'history' | 'hash''history'History mode uses clean URLs (/about) and requires server-side fallback. Hash mode uses #/about and works on any static host.
basestring''Base path prefix for sub-directory deployments. Auto-detected from <base href> if not set (also checks window.__ZQ_BASE). Trailing slash is removed.
routesArray[]Array of route definitions (see below)
fallbackstring | nullnullComponent name to mount when no route matches (404 page)
Tip: Use hash mode if you're deploying to a static host (GitHub Pages, S3, file://) where you can't configure server-side URL rewriting. History mode gives cleaner URLs but requires a catch-all rewrite rule on the server.
file:// auto-detection: When the page is opened from disk (via file:// protocol), the router automatically switches to hash mode regardless of configuration. This enables the CLI bundler output to work without any server.

Route Definition Object

PropertyTypeRequiredDescription
pathstringYesURL pattern. Use :param for dynamic segments (e.g. /user/:id) and * for wildcards (e.g. /docs/*).
componentstring | functionYesRegistered component name, or a render function (route) => htmlString for inline rendering.
load() => PromiseNoLazy-load function called before mounting. Use for code-splitting: () => import('./heavy-page.js')
fallbackstringNoAn additional path that also matches this route. Useful when a route has a dynamic :param segment but you also want the bare path to render the same component (with the param undefined).
// Examples of route definitions:
{ path: '/',              component: 'home-page' }            // exact match
{ path: '/user/:id',      component: 'user-page' }            // dynamic param
{ path: '/user/:id/posts', component: 'user-posts' }          // multi-segment
{ path: '/docs/:section', component: 'docs-page',             // fallback path
                           fallback: '/docs' }
{ path: '/docs/*',         component: 'docs-page' }           // wildcard
{ path: '/lazy',           component: 'lazy-page',            // lazy-loaded
                            load: () => import('./lazy.js') }

Navigation

MethodSignatureDescription
navigatenavigate(path, options?)Push a new entry to history and resolve. options.state sets history.state.
replacereplace(path, options?)Replace current history entry (no back button entry). options.state supported.
backback()history.back()
forwardforward()history.forward()
gogo(n)history.go(n) — negative values go back, positive go forward
addadd(route)Dynamically add a route definition at runtime
removeremove(path)Remove a route by path string
destroydestroy()Tear down the router: destroy mounted component, clear routes/guards/listeners
router.navigate('/user/42');                    // push
router.navigate('/user/42', { state: { from: 'list' } }); // push with state
router.replace('/login');                       // replace (no history entry)
router.back();
router.go(-2);

// Dynamic routes
router.add({ path: '/new', component: 'new-page' });
router.remove('/old');

Navigation Links — z-link

Use the z-link attribute on anchor tags for SPA navigation. The router intercepts clicks on these elements and calls navigate() instead of triggering a full page reload.

<a z-link="/">Home</a>
<a z-link="/user/42">Profile</a>
<a z-link="/settings?tab=general">Settings</a>
Active link styling: The router adds an .active class to [z-link] elements whose path matches the current route. Use this in your CSS to highlight the current page in the navigation.

Route Params & Query in Components

When the router mounts a component, it passes special props containing route information:

PropTypeDescription
this.props.$paramsObjectDynamic route parameters (e.g. { id: '42' } from /user/:id)
this.props.$queryObjectParsed query string (e.g. { tab: 'general' } from ?tab=general)
this.props.$routeObjectFull route object: { route, params, query, path }
this.props.{param}stringParams are also spread as top-level props (e.g. this.props.id)
$.component('user-page', {
  render() {
    const id = this.props.$params.id;       // or this.props.id
    const tab = this.props.$query.tab;      // query string
    const route = this.props.$route;        // full route info
    return `
      <h1>User ${id}</h1>
      <p>Tab: ${tab || 'default'}</p>
      <p>Path: ${route.path}</p>
    `;
  }
});

Navigation Guards

Guards let you intercept navigation to protect routes, redirect, or run side effects.

GuardSignatureDescription
beforeEachbeforeEach(fn(to, from))Runs before route change. Return false to cancel, return a path string to redirect, or void to proceed.
afterEachafterEach(fn(to, from))Runs after route change. Good for analytics, scroll restoration, etc.
// Protect a route — redirect to login if not authenticated
router.beforeEach((to, from) => {
  if (to.path === '/admin' && !isLoggedIn()) {
    return '/login';   // redirect
  }
  // return false to cancel navigation entirely
});

// Analytics tracking after navigation
router.afterEach((to, from) => {
  analytics.track(to.path);
  window.scrollTo(0, 0);  // scroll to top on route change
});
Guard parameters: Both to and from are route objects with shape { route, params, query, path }. On the very first navigation, from will be null.

Change Listener

Subscribe to route changes without using a guard. Returns an unsubscribe function.

const unsub = router.onChange((to, from) => {
  console.log(from?.path, '→', to.path);
  document.title = to.route?.title || 'My App';
});

// Stop listening
unsub();

Current Route State

PropertyReturnsDescription
router.current{ route, params, query, path } | nullFull current route state
router.pathstringCurrent path (base-stripped in history mode)
router.queryObjectCurrent parsed query parameters
// Check current route programmatically
if (router.path === '/admin') { /* ... */ }
console.log(router.query);   // { page: '2', sort: 'name' }
console.log(router.current); // full route info
Tip: Use $.getRouter() to access the active router instance from anywhere (e.g. inside a component method or a utility module) without importing the variable directly.
