Getting Started

zQuery is a lightweight, zero-dependency frontend library (~45 KB minified). It provides a jQuery-inspired selector API, reactive components, an SPA router, global state management, an HTTP client, and utility helpers — all from a single script tag. Works out of the box with standard ES modules — no npm install or transpiler required. An optional CLI bundler is available for single-file distribution.

Quick Start (ES Modules — Recommended)

The preferred way to use zQuery is with the pre-built browser bundle (zQuery.min.js) loaded via a <script> tag, with your app code served as standard ES modules. No npm install, no transpiler, no build step. Grab the latest build from GitHub and drop it into your project:

Download from GitHub Download dist/zquery.min.js from the repo, or clone and run node build.js

1. Get the Library

git clone https://github.com/tonywied17/zero-query.git
cd zero-query
node build.js
# Creates dist/zQuery.js     (~78 KB, readable)
# Creates dist/zQuery.min.js (~45 KB, production)

The build script concatenates all source modules into a self-contained IIFE that exposes $ and zQuery on the window object. Copy dist/zQuery.min.js into your project's scripts/vendor/ folder.

2. Include in HTML

<!-- Load zQuery globally ($ and zQuery available on window) -->
<script src="scripts/vendor/zQuery.min.js"></script>

<!-- Your app code as ES module -->
<script type="module" src="scripts/app.js"></script>
Load order matters: The zQuery.min.js script tag must appear before your module scripts so that $ is available globally when your components execute.

3. Boot Your App

// scripts/app.js
import './components/home.js';
import './components/about.js';
import { routes } from './routes.js';

// Global styles are loaded via <link rel="stylesheet"> in index.html (recommended).
// $.style() is available for dynamically loading additional CSS files at runtime.

const router = $.router({
  el: '#app',
  routes,
  fallback: 'not-found',
});

$.ready(() => {
  console.log('zQuery v' + $.version + ' loaded');
});

The app.js file is the single entry point: it imports all component files, defines routes, creates the store (if needed), and boots the router. Component files register themselves via $.component(), so importing them is enough.

4. Define a Component

// scripts/components/home.js
$.component('home-page', {
  state: () => ({ greeting: 'Hello, World!' }),

  render() {
    return `
      <div class="card">
        <h2>${this.state.greeting}</h2>
        <button @click="update">Change Greeting</button>
      </div>
    `;
  },

  update() {
    this.state.greeting = 'Welcome to zQuery!';
  }
});
Tip: Component names must contain a hyphen (e.g. home-page, app-counter). This follows the HTML custom element convention and prevents collisions with native tags.

ES Module Imports (Advanced)

If you prefer ES modules over the global $, the source entry (index.js) exports everything as named exports:

import { $, zQuery, ZQueryCollection, queryAll,
  reactive, signal, computed, effect,
  component, mount, mountAll, getInstance, destroy,
  createRouter, getRouter, createStore, getStore,
  http, debounce, throttle, pipe, once, sleep,
  escapeHtml, html, trust, uuid, deepClone, deepMerge,
  isEqual, param, parseQuery, storage, session, bus
} from './index.js';
Key point: Most apps should use the built bundle (zQuery.min.js) which exposes $ and zQuery globally. Your component files use $ directly — no imports needed for the library itself. The ES module exports are available for advanced setups or tree-shaking bundlers.

CLI Bundler (Optional — Beta)

zQuery includes a zero-dependency CLI tool that can compile your entire app — all ES modules, the library, external templates, and static assets — into a single bundled file. The bundler is provided as an alternative to the ES module setup for use cases like:

  • Distributing an app as one .html + one .js file
  • Deploying to environments without a web server (open index.html from disk via file://)
  • Reducing HTTP requests on legacy hosting without HTTP/2
Beta notice: The CLI bundler is functional but still maturing. The ES module setup above is the recommended and most reliable approach. Use the bundler when you need single-file distribution or offline file:// support.

Installing the CLI

The CLI is included in the zero-query npm package. Install it to get the zquery command along with the library source and pre-built bundles:

# Recommended: add as a dev dependency
npm install zero-query --save-dev
npx zquery bundle

# Or install globally
npm install -g zero-query
zquery bundle

# Or run once without installing (npx fetches it on demand)
npx zero-query bundle

The package includes:

  • dist/zQuery.min.js — pre-built browser bundle (copy into your project's vendor/ folder)
  • cli.js — the CLI tool (zquery build / zquery bundle)
  • src/ — library source modules
Quick setup: For a new project, run npm init -y then npm install zero-query --save-dev. Copy the pre-built library with cp node_modules/zero-query/dist/zQuery.min.js scripts/vendor/. You're ready to use both the ES module setup and the CLI bundler.

Commands

zquery build — Build the Library

Compiles the zQuery library source into dist/zQuery.js and dist/zQuery.min.js. Equivalent to node build.js.

zquery build                # one-time build
zquery build --watch        # rebuild on source changes

zquery bundle [entry] — Bundle an App

Walks the ES module import graph starting from an entry file, topologically sorts all dependencies, strips import/export syntax, and concatenates everything into a single IIFE.

# Auto-detect entry from index.html
zquery bundle

# Specify entry explicitly
zquery bundle scripts/app.js

# Full example with all options
zquery bundle scripts/app.js \
  --out dist/app.bundle.js \
  --include-lib \
  --html index.html \
  --watch

Bundle Options

FlagShortDescription
--out <path>-oOutput file path. Default: dist/<entry>.bundle.js
--include-lib-LEmbed zquery.min.js directly in the bundle (no separate script tag needed)
--html <file>—Rewrite the given HTML file: replaces module script with bundle, copies assets into dist/, adjusts <base href>
--watch-wWatch source files and rebuild automatically on changes

What the Bundler Does

  1. Import graph walking — Recursively resolves all import statements (including side-effect import './foo.js') and produces a topologically sorted dependency list (leaves first).
  2. Module syntax stripping — Removes import/export keywords while preserving all declarations. The output is plain browser-compatible JavaScript.
  3. IIFE wrapping — The concatenated code is wrapped in (function() { 'use strict'; ... })() to avoid polluting the global scope.
  4. Library embedding (--include-lib) — Finds zquery.min.js in common locations (scripts/vendor/, dist/, etc.) and embeds it at the top of the bundle. The result is fully self-contained.
  5. External resource inlining — Automatically detects pages configs, templateUrl, and styleUrl references in your components and inlines the referenced HTML/CSS files. This enables file:// support where fetch() would otherwise fail due to CORS.
  6. HTML rewriting (--html) — Replaces <script type="module"> with <script defer>, removes the standalone library tag when embedded, rewrites <base href="/"> to <base href="./">, and copies all referenced assets (CSS, images, vendor JS) into dist/.
  7. Minification — Produces both a readable .bundle.js and a minified .bundle.min.js.

Entry Detection

When no entry file is provided, the bundler auto-detects it:

  1. Reads index.html (or public/index.html) and looks for <script type="module" src="...">
  2. Falls back to common file paths: scripts/app.js, src/app.js, js/app.js, app.js, main.js

Automatic Transformations

SourceTransformation
import ... from './x.js'Removed (code is concatenated in dependency order)
import './x.js'Removed (side-effect import)
export default ...export default keyword removed, declaration kept
export const/function/classexport keyword removed, declaration kept

import.meta.urlReplaced with new URL('<relative-path>', document.baseURI).href

Hash Routing on file://

When the bundled app is opened via file://, the router automatically detects the protocol and switches to hash-based routing (#/about instead of /about). No configuration needed — z-link attributes and programmatic navigation work identically.

Preparing Your App for Bundling

The bundler works with the standard zQuery project structure out of the box. Keep these points in mind:

  • Use relative imports — import './components/home.js' (not bare specifiers like import 'home')
  • One component per file — The import graph walker resolves each file once and sorts them by dependency
  • import.meta.url — Automatically replaced at bundle time; no manual changes needed
  • External templates — templateUrl, styleUrl, and pages configs are automatically detected and inlined into the bundle
  • Vendor scripts — With --include-lib, the bundler searches scripts/vendor/, vendor/, lib/, and dist/ for zquery.min.js

Step-by-Step: Bundling Your Own App

# 1. Install zero-query in your project
cd my-app
npm init -y                            # skip if you already have a package.json
npm install zero-query --save-dev

# 2. Copy the pre-built library into your project
cp node_modules/zero-query/dist/zQuery.min.js scripts/vendor/

# 3. Bundle the app
npx zquery bundle scripts/app.js -o dist/app.bundle.js -L --html index.html

# Output:
#   dist/index.html          ← rewritten HTML
#   dist/app.bundle.js       ← full bundle (library + app + inlined templates)
#   dist/app.bundle.min.js   ← minified version
#   dist/styles/             ← copied CSS
#   dist/scripts/vendor/     ← copied vendor assets

# 4. Open directly in a browser — no server needed
start dist/index.html

Bundling the Starter App (from source)

If you cloned the zero-query repository directly:

node build.js
cp dist/zQuery.min.js examples/starter-app/scripts/vendor/
cd examples/starter-app
npx zquery bundle scripts/app.js -o dist/app.bundle.js -L --html index.html
start dist/index.html
Tip: Use watch mode (-w) during development to automatically rebuild when you save changes: npx zquery bundle scripts/app.js -o dist/app.bundle.js -L --html index.html -w
