Getting Started

zQuery is a lightweight, zero-dependency frontend library (~54 KB minified). It provides a jQuery-inspired selector API, reactive components, an SPA router, global state management, an HTTP client, and utility helpers — all from a single script tag. Works out of the box with standard ES modules. An optional CLI dev server with live-reload and a production bundler make the full development cycle seamless.

Quick Start (CLI Dev Server — Recommended)

The fastest way to develop with zQuery is via the built-in CLI dev server. It serves your ES modules as-is with live-reload, CSS hot-swap, and SPA fallback routing — and automatically resolves the library so you never have to manually download or copy zQuery.min.js.

1. Install

# Per-project (recommended)
npm install zero-query --save-dev

# Or install globally to use zquery anywhere without npx
npm install zero-query -g

2. Create Your Project

Scaffold a ready-to-run project with the create command:

npx zquery create my-app

This generates a fully-featured starter app with a router, global store, seven demo components, a responsive sidebar layout, and a stylesheet — everything you need to start building. The resulting structure looks like this:

  • ▶my-app
    • index.html
    • ▶scripts
      • app.js
      • routes.js
      • store.js
      • ▶components
        • home.js
        • counter.js
        • todos.js
        • api-demo.js
        • about.js
        • not-found.js
        • ▶contactsfolder component (templateUrl + styleUrl)
          • contacts.js
          • contacts.html
          • contacts.css
    • ▶styles
      • styles.css
What’s included: The scaffold wires up a global store (store.js) with todos and a visit counter, six routed pages (/, /counter, /todos, /contacts, /api, /about), a 404 fallback component, and a responsive sidebar with mobile menu. The pages demonstrate $.store, $.router, $.http, $.bus, reactive state, template directives (z-if, z-for, z-show, z-class, z-model, z-text), and external templates via templateUrl / styleUrl.
Tip: You can also run npx zquery create (without a directory name) to scaffold into the current directory.

3. Start the Dev Server

npx zquery dev my-app

# Custom port
npx zquery dev my-app --port 8080

The server starts watching for changes and the library is served automatically — no need to place zQuery.min.js anywhere in your project. Just reference it in your HTML as shown below and the dev server handles the rest.

Tip: Stay in the project root (where node_modules lives) instead of cd-ing into your app folder. This keeps index.d.ts accessible to your IDE for full type / IntelliSense support.
Tip: See the Development Workflow page for full details on live-reload, CSS hot-swap, and SSE.

Alternative: Manual Setup (No npm Required)

If you prefer a standalone setup, you can download the pre-built library and open your project directly in the browser — just a script tag and you're ready:

Download from GitHub Download dist/zquery.min.js from the repo, or clone and run npx zquery build

Place the file at scripts/vendor/zQuery.min.js in your project, then open index.html in your browser (or use any static file server).

Note: The manual approach works great for quick prototypes and learning. For active development, the CLI dev server is highly recommended — it gives you live-reload, automatic library resolution, and SPA routing out of the box.

Include in HTML

<!-- Load zQuery globally ($ and zQuery available on window) -->
<script src="scripts/vendor/zQuery.min.js"></script>

<!-- Your app code as ES module -->
<script type="module" src="scripts/app.js"></script>
Load order matters: The zQuery.min.js script tag must appear before your module scripts so that $ is available globally when your components execute.
Dev server users: The scripts/vendor/zQuery.min.js path still works as the HTML reference — the dev server automatically intercepts the request and serves the latest library version from your npm install. No file needs to exist at that path during development. To disable this and serve your own local copy instead, start the server with --no-intercept.

Boot Your App

// scripts/app.js
import './components/home.js';
import './components/about.js';
import './components/contacts/contacts.js';
import { routes } from './routes.js';

// Global styles are loaded via <link rel="stylesheet"> in index.html (recommended).
// $.style() is available for dynamically loading additional CSS files at runtime.

const router = $.router({
  el: '#app',
  routes,
  fallback: 'not-found',
});

$.ready(() => {
  console.log('zQuery v' + $.version + ' loaded');
});

The app.js file is the single entry point: it imports all component files, defines routes, creates the store (if needed), and boots the router. Component files register themselves via $.component(), so importing them is enough.

Define a Component

// scripts/components/home.js
$.component('home-page', {
  state: () => ({ greeting: 'Hello, World!' }),

  render() {
    return `
      <div class="card">
        <h2>${this.state.greeting}</h2>
        <button @click="update">Change Greeting</button>
      </div>
    `;
  },

  update() {
    this.state.greeting = 'Welcome to zQuery!';
  }
});
Tip: Component names must contain a hyphen (e.g. home-page, app-counter). This follows the HTML custom element convention and prevents collisions with native tags.

ES Module Imports (Advanced)

If you prefer ES modules over the global $, the source entry (index.js) exports everything as named exports:

import { $, zQuery, ZQueryCollection, queryAll,
  reactive, signal, computed, effect,
  component, mount, mountAll, getInstance, destroy, getRegistry, style,
  createRouter, getRouter, createStore, getStore,
  http, debounce, throttle, pipe, once, sleep,
  escapeHtml, html, trust, uuid, camelCase, kebabCase,
  deepClone, deepMerge, isEqual, param, parseQuery,
  storage, session, bus
} from './index.js';
Key point: Most apps should use the built bundle (zQuery.min.js) which exposes $ and zQuery globally. Your component files use $ directly — no imports needed for the library itself. The ES module exports are available for advanced setups or tree-shaking bundlers.
