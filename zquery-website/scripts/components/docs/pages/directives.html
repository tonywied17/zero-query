Directives

Directives are special attributes in your templates that zQuery processes after each render. They provide declarative DOM manipulation — conditional rendering, list iteration, dynamic attributes, and more — without writing imperative JavaScript. Directives work in both render() template literals and external templateUrl HTML files.

Naming convention: All built-in directives use the z- prefix (e.g. z-if, z-for, z-show). Event binding uses either @event or z-on:event syntax. The : shorthand (e.g. :href) is an alias for z-bind:.

Quick Reference

z-else-if<div z-else-if="loading">Else-if branch (must follow z-if or z-else-if)
z-else<div z-else>Default branch (must follow z-if or z-else-if)
z-for<li z-for="item in items">List rendering — repeats element for each item
z-show<div z-show="visible">Toggle display: none (element stays in DOM)
z-bind: / :<a :href="url">Dynamic attribute binding

Conditional Rendering — z-if / z-else-if / z-else

Conditionally include or exclude elements from the DOM based on an expression. Unlike z-show, elements removed by z-if are fully removed from the document — they don’t exist in the DOM at all.

<!-- Simple conditional -->
<div z-if="items.length > 0">
  <p>You have items!</p>
</div>
<div z-else>
  <p>No items yet.</p>
</div>
<!-- Multi-branch chain -->
<div z-if="status === 'loading'">Loading…</div>
<div z-else-if="status === 'error'">Something went wrong.</div>
<div z-else-if="status === 'empty'">No results found.</div>
<div z-else>Showing results</div>

In a Component

$.component('status-card', {
  state: () => ({ status: 'loading', data: null }),

  render() {
    return `
      <div z-if="status === 'loading'">
        <span class="spinner"></span> Loading…
      </div>
      <div z-else-if="status === 'error'">
        <p class="error">Failed to load.</p>
        <button @click="retry">Retry</button>
      </div>
      <div z-else>
        <h2>${this.state.data?.title}</h2>
      </div>
    `;
  },

  retry() { this.state.status = 'loading'; /* ... */ }
});
Tip: z-else-if and z-else elements must be immediate siblings of the preceding z-if or z-else-if element — no other elements in between. The expression is evaluated in the component’s state context, so z-if="count > 0" reads this.state.count.

z-if vs z-show

Both directives hide content conditionally, but they use fundamentally different mechanisms. Choosing the right one affects performance and behaviour:

z-ifz-show
DOM behaviourElement is removed when falsyElement stays in DOM, display: none applied
PerformanceBetter for rarely-toggled contentBetter for frequently-toggled content (avoids DOM add/remove)
Supports else chainYes (z-else-if, z-else)No
Initial render costNothing rendered if falsy on first paintElement is always rendered, just hidden
Use whenContent may never be shown, or has expensive childrenContent toggles often (tabs, dropdowns, tooltips)

Side-by-Side Example

<!-- z-if: element removed entirely when logged out -->
<div z-if="isLoggedIn">
  <p>Welcome, {{user.name}}</p>
  <button @click="logout()">Sign out</button>
</div>

<!-- z-show: element stays in DOM, just hidden via CSS -->
<div z-show="menuOpen" class="dropdown">
  <a z-link="/settings">Settings</a>
  <a z-link="/profile">Profile</a>
</div>
Rule of thumb: Use z-if for content that appears once (login gates, permission checks, feature flags). Use z-show for UI that toggles frequently (menus, accordions, modal overlays) where the DOM cost of re-creating elements would be wasteful.

List Rendering — z-for

Repeat an element once for each item in an array (or object, or number range). Inner content uses {{expression}} syntax to reference the current item and index.

Basic Array Iteration

<ul>
  <li z-for="item in items">{{item.name}} — ${{item.price}}</li>
</ul>

With Index

<ol>
  <li z-for="(item, i) in items">{{i + 1}}. {{item.name}}</li>
</ol>

Number Range

<!-- z-for="n in 5" iterates [1, 2, 3, 4, 5] -->
<div class="stars">
  <span z-for="n in 5">★</span>
</div>

Object Iteration

<!-- Iterating an object produces {key, value} entries -->
<dl z-for="(entry, i) in settings">
  <dt>{{entry.key}}</dt>
  <dd>{{entry.value}}</dd>
</dl>

Complete z-for Component Example

$.component('todo-list', {
  state: () => ({
    newTodo: '',
    todos: [
      { text: 'Learn zQuery', done: true },
      { text: 'Build something', done: false },
    ]
  }),

  render() {
    return `
      <input z-model="newTodo" placeholder="Add todo…" @keydown.prevent="addOnEnter">
      <ul>
        <li z-for="(todo, i) in todos"
            class="{{todo.done ? 'done' : ''}}">
          {{todo.text}}
          <button @click="remove(${'{{'} i {{'}}'}})">×</button>
        </li>
      </ul>
      <p z-if="!todos.length">All done! </p>
    `;
  },

  addOnEnter(e) {
    if (e.key !== 'Enter' || !this.state.newTodo.trim()) return;
    this.state.todos = [
      ...this.state.todos.__raw,
      { text: this.state.newTodo.trim(), done: false }
    ];
    this.state.newTodo = '';
  },

  remove(index) {
    this.state.todos = this.state.todos.__raw.filter((_, i) => i !== index);
  }
});
How z-for works: zQuery expands z-for elements before inserting into the DOM. The {{expression}} syntax inside the element is evaluated once per item with the item variable in scope. This means {{item.name}} resolves to each item’s name property. In render() templates, use {{}} (double braces) for z-for item expressions — ${} is evaluated by JavaScript before zQuery sees it.
Tip: For complex rendering logic, you can always use native JavaScript .map().join('') in your render() function. z-for is a convenience for simple iteration, especially in external templateUrl HTML files where JavaScript isn’t available.

Toggle Visibility — z-show

Toggle an element’s visibility by setting display: none. Unlike z-if, the element remains in the DOM — it’s just hidden.

<div z-show="isOpen">
  <p>Dropdown content</p>
</div>

<!-- Common pattern: loading overlay -->
<div class="overlay" z-show="loading">Loading…</div>
$.component('toggle-panel', {
  state: () => ({ open: false }),
  render() {
    return `
      <button @click="toggle">
        ${this.state.open ? 'Hide' : 'Show'} Panel
      </button>
      <div class="panel" z-show="open">
        <p>Panel content here</p>
      </div>
    `;
  },
  toggle() { this.state.open = !this.state.open; }
});

Dynamic Attributes — z-bind: / :

Dynamically bind any HTML attribute to an expression. Use the full z-bind:attr syntax or the shorthand :attr.

<!-- Full syntax -->
<a z-bind:href="url">Link</a>
<img z-bind:src="imageUrl" z-bind:alt="imageAlt">

<!-- Shorthand (recommended) -->
<a :href="url">Link</a>
<img :src="imageUrl" :alt="imageAlt">

<!-- Boolean attributes: false/null/undefined removes the attribute -->
<button :disabled="isSubmitting">Submit</button>
<input :readonly="!canEdit">
$.component('dynamic-link', {
  state: () => ({
    url: 'https://z-query.com',
    external: true,
    disabled: false,
  }),
  render() {
    return `
      <a :href="url"
         :target="external ? '_blank' : null"
         :rel="external ? 'noopener' : null">
        Visit Site
      </a>
      <button :disabled="disabled" @click="navigate">Go</button>
    `;
  },
  navigate() { window.location.href = this.state.url; }
});
Behaviour with special values: false, null, and undefined remove the attribute entirely (important for boolean attributes like disabled, readonly, hidden). true sets the attribute with an empty value (disabled="").

Dynamic Classes — z-class

Add CSS classes dynamically using a string, array, or object map. Merges with any existing class attribute on the element.

<!-- Object syntax: keys are class names, values are booleans -->
<div z-class="{ active: isActive, 'text-danger': hasError }">

<!-- String syntax -->
<div z-class="currentTheme">

<!-- Array syntax -->
<div z-class="['base', isLarge ? 'lg' : 'sm']">
$.component('tab-button', {
  state: () => ({ activeTab: 'home' }),
  render() {
    const tabs = ['home', 'about', 'contact'];
    return tabs.map(tab => `
      <button class="tab"
              z-class="{ active: activeTab === '${tab}' }"
              @click="setTab('${tab}')">
        ${tab}
      </button>
    `).join('');
  },
  setTab(tab) { this.state.activeTab = tab; }
});

Dynamic Styles — z-style

Apply inline CSS styles dynamically using a string or object map.

<!-- Object syntax: camelCase or quoted kebab-case keys -->
<div z-style="{ color: textColor, fontSize: size + 'px' }">

<!-- String syntax -->
<div z-style="'background:' + bgColor + ';opacity:' + alpha">
$.component('color-picker', {
  state: () => ({ hue: 200, size: 80 }),
  render() {
    return `
      <div class="preview"
           z-style="{ backgroundColor: 'hsl(' + hue + ', 70%, 50%)',
                      width: size + 'px', height: size + 'px' }">
      </div>
      <input z-model="hue" type="range" min="0" max="360">
      <input z-model="size" type="range" min="20" max="200">
    `;
  }
});

Content Directives — z-html & z-text

Set an element’s content from a state expression. z-text uses safe textContent (no HTML parsing), while z-html uses innerHTML.

<!-- z-text: safe — HTML entities are escaped -->
<span z-text="userName"></span>

<!-- z-html: raw HTML injection (use with trusted content only) -->
<div z-html="markdownOutput"></div>
$.component('article-view', {
  state: () => ({
    title: 'Hello <World>',
    body: '<p>This is <strong>rich</strong> content.</p>',
  }),
  render() {
    return `
      <!-- Title is safely escaped — no XSS risk -->
      <h2 z-text="title"></h2>

      <!-- Body is injected as raw HTML (trusted source only) -->
      <div class="body" z-html="body"></div>
    `;
  }
});
Security: z-html injects raw HTML. Never use it with user-supplied content without sanitizing first. Use z-text for user input — it escapes HTML automatically.

Event Binding — @event / z-on:event

Bind DOM events to component methods with delegated event handling. Both @event and z-on:event syntax are supported (they are identical in behaviour).

<!-- Shorthand (recommended) -->
<button @click="save">Save</button>

<!-- Full syntax -->
<button z-on:click="save">Save</button>

<!-- With arguments -->
<button @click="remove(3)">Remove</button>
<button @click="setFilter('active')">Active</button>

<!-- Access the native event -->
<input @input="search($event)">

Event Modifiers

Append modifiers after the event name with dots. Multiple modifiers can be chained:

ModifierSyntaxEffect
.prevent@submit.prevent="save"Calls event.preventDefault()
.stop@click.stop="toggle"Calls event.stopPropagation()
.self@click.self="close"Only fires if event.target is the element itself (not a child)
.once@click.once="init"Handler fires at most once per element
.capture@click.capture="log"Uses capture phase (addEventListener with {capture: true})
.passive@scroll.passive="track"Registers a passive listener (improves scroll performance)
.debounce.{ms}@input.debounce.300="search"Debounce — delays invocation until {ms} of inactivity (default 250 ms)
.throttle.{ms}@scroll.throttle.100="update"Throttle — invokes at most once per {ms} interval (default 250 ms)
 
<!-- Prevent form submission + call method -->
<form @submit.prevent="handleSubmit">

<!-- Debounce search input by 300ms -->
<input @input.debounce.300="search" placeholder="Search…">

<!-- Throttle scroll tracking to 100ms -->
<div class="feed" @scroll.throttle.100="trackScroll">

<!-- Close modal only when clicking the backdrop (not inner content) -->
<div class="modal-backdrop" @click.self="close">
  <div class="modal">…</div>
</div>

<!-- Initialize once, then ignore subsequent clicks -->
<button @click.once="loadMore">Load More</button>

<!-- Chain modifiers -->
<a @click.prevent.stop="navigate">Link</a>

Two-Way Binding — z-model

z-model creates a reactive two-way sync between a form element and a state property. See the Components documentation for full coverage including supported elements, nested keys, and modifiers (z-lazy, z-trim, z-number).

<input z-model="name" placeholder="Your name">
<input z-model="age" type="number">
<input z-model="agreed" type="checkbox">
<select z-model="color">
  <option value="red">Red</option>
  <option value="blue">Blue</option>
</select>
<textarea z-model="notes" z-lazy></textarea>

Element References — z-ref

Capture a direct reference to a DOM element so you can access it in your methods via this.refs.name. Refs are refreshed after every render.

$.component('focus-input', {
  state: () => ({ query: '' }),
  render() {
    return `
      <input z-ref="searchInput" z-model="query" placeholder="Search">
      <button @click="focusInput">Focus</button>
    `;
  },
  focusInput() {
    this.refs.searchInput.focus();
  }
});

Prevent Flash — z-cloak

Elements with z-cloak are hidden (via display: none !important) until the component renders and removes the attribute. This prevents users from seeing raw template syntax like {{expression}} before the component hydrates.

<!-- Hidden until the component renders -->
<div z-cloak>
  {{greeting}}
</div>
Tip: zQuery automatically injects a global <style> that includes [z-cloak] { display: none !important } and -webkit-tap-highlight-color: transparent on *, *::before, *::after for mobile tap-highlight suppression. You don’t need to add either rule yourself.

Skip Processing — z-pre

Tells zQuery to skip all directive processing for this element and its descendants. Useful for displaying raw template syntax in documentation or code examples.

<!-- This content is NOT processed — directives are preserved as-is -->
<div z-pre>
  <span z-if="show">This z-if is not evaluated</span>
  <span z-for="item in list">{{item}}</span>
</div>

Expression Context

All directive expressions are evaluated in the component’s state context. This means bare property names resolve to this.state.*. You also have access to props and refs:

ExpressionResolves To
countthis.state.count
items.length > 0this.state.items.length > 0
props.titlethis.props.title
refs.inputthis.refs.input
true / 42 / 'hello'Literal values
External templates: In templateUrl HTML files, directives work exactly the same way. The {{expression}} interpolation and z-for / z-if directives are fully supported, making external templates just as powerful as inline render() functions.

Directives in External Templates

Directives are especially useful in templateUrl HTML files, where you don’t have access to JavaScript template literal expressions. The full directive set is available:

<!-- template.html (loaded via templateUrl) -->
<div class="dashboard">
  <h2 z-text="title"></h2>

  <div z-if="loading">Loading…</div>
  <div z-else>
    <ul>
      <li z-for="item in items">
        {{item.name}} — ${{item.price.toFixed(2)}}
      </li>
    </ul>
  </div>

  <p z-show="items.length === 0">No items found.</p>
  <button :disabled="loading" z-on:click="refresh">Refresh</button>
</div>
// component.js
$.component('my-dashboard', {
  state: () => ({ title: 'Dashboard', loading: false, items: [] }),
  templateUrl: 'template.html',

  mounted() { this.refresh(); },

  async refresh() {
    this.state.loading = true;
    const res = await $.get('/api/items');
    this.state.items = res.data;
    this.state.loading = false;
  }
});

Processing Order

Directives are processed in a specific order during each render cycle:

  1. z-for — list expansion (pre-DOM, string level)
  2. z-pre — skip directive processing (subtree exempt)
  3. z-if / z-else-if / z-else — conditional removal
  4. z-show — visibility toggle
  5. z-bind: / : — dynamic attributes
  6. z-class — dynamic classes
  7. z-style — dynamic styles
  8. z-html — HTML injection
  9. z-text — safe text binding
  10. z-cloak — attribute removal
  11. @event / z-on: — event binding
  12. z-ref — element references
  13. z-model — two-way binding
Performance note: Unlike frameworks that use a virtual DOM, zQuery processes directives as a single pass over the real DOM after each innerHTML update. This is fast for typical component sizes. For very large lists (1000+ items), consider using .map().join('') in render() for maximum performance.
