HTTP Client

A lightweight wrapper around the native fetch() API with automatic JSON handling, request/response interceptors, configurable base URL, timeout and abort support, and a clean result object.

Request Methods

MethodSignatureDescription
$.get$.get(url, params?, opts?)GET request. params object is appended as query string.
$.post$.post(url, data?, opts?)POST request. data is JSON-stringified (unless FormData).
$.put$.put(url, data?, opts?)PUT request.
$.patch$.patch(url, data?, opts?)PATCH request.
$.delete$.delete(url, data?, opts?)DELETE request.
// GET with query params — appended as ?page=1&limit=10
const res = await $.get('/api/users', { page: 1, limit: 10 });
console.log(res.data);   // parsed JSON body
console.log(res.status); // 200

// POST with JSON body
await $.post('/api/users', { name: 'Tony', email: 'tony@x.com' });

// PUT, PATCH, DELETE
await $.put('/api/users/1', { name: 'Updated' });
await $.patch('/api/users/1', { email: 'new@x.com' });
await $.delete('/api/users/1');

Configuration

Set defaults that apply to every request:

$.http.configure({
  baseURL: 'https://api.example.com',   // prepended to all non-absolute URLs
  headers: { Authorization: 'Bearer token' }, // merged with per-request headers
  timeout: 15000,  // ms — default is 30000 (30 seconds)
});
OptionTypeDefaultDescription
baseURLstring''Prepended to URLs that don't start with http
headersObject{ 'Content-Type': 'application/json' }Default headers for all requests (merged with per-request headers)
timeoutnumber30000Request timeout in milliseconds. Set 0 for no timeout.
Tip: Per-request options override the global config. You can pass custom headers, a different timeout, or an AbortController signal as the third argument to any request method: $.get(url, params, { headers: { ... }, timeout: 5000 })

Response Object

Every request resolves with a standardized result object:

PropertyTypeDescription
okbooleanresponse.ok — true if status is 200-299
statusnumberHTTP status code (200, 404, 500, etc.)
statusTextstringHTTP status text ('OK', 'Not Found', etc.)
headersObjectResponse headers as a plain object
dataanyAuto-parsed body: JSON, text, or blob based on Content-Type
responseResponseRaw fetch Response object for advanced use
Auto-parsing: The response body is parsed automatically based on the Content-Type header: application/json → JSON, text/* → string, binary types → Blob. If no Content-Type is present, it tries JSON first, then falls back to text.

Interceptors

Interceptors let you modify requests before they're sent and process responses globally.

// Request interceptor — modify headers, URL, or block the request
$.http.onRequest((fetchOpts, url) => {
  fetchOpts.headers['X-Request-ID'] = $.uuid();

  // Return false to BLOCK the request entirely
  // Return { url, options } to modify the URL or fetch options
  // Return void/undefined to proceed unchanged
});

// Response interceptor — global error handling, token refresh, etc.
$.http.onResponse((result) => {
  if (result.status === 401) {
    window.location = '/login';
  }
});
InterceptorCallback SignatureCan Return
onRequest(fetchOpts, url) => ...void (proceed), false (block), or { url?, options? } (modify)
onResponse(result) => voidNothing (read-only)

Abort / Cancel

Cancel in-flight requests using the AbortController pattern:

const controller = $.http.createAbort();  // shorthand for new AbortController()
$.get('/api/slow', null, { signal: controller.signal });
setTimeout(() => controller.abort(), 2000);  // cancel after 2s

// You can also pass your own AbortController:
const ac = new AbortController();
$.get('/api/data', null, { signal: ac.signal });

FormData Upload

When the request body is FormData, the Content-Type header is automatically removed so the browser can set the correct multipart/form-data boundary.

const fd = new FormData();
fd.append('file', fileInput.files[0]);
fd.append('name', 'document.pdf');
await $.post('/api/upload', fd);  // Content-Type set by browser

Error Handling

Non-2xx responses throw an Error with a .response property containing the full result object. Timeout errors throw with a descriptive message.

try {
  const res = await $.get('/api/data');
} catch (err) {
  if (err.response) {
    // HTTP error (4xx, 5xx)
    console.log(err.response.status);     // 404, 500, etc.
    console.log(err.response.data);       // error body (parsed)
  } else {
    // Network error or timeout
    console.log(err.message);
    // e.g. "Request timeout after 30000ms: GET https://..."
  }
}

Raw Fetch Passthrough

If you need the raw Response (e.g. for streaming), bypass the wrapper:

const raw = await $.http.raw('/api/stream');
const reader = raw.body.getReader();

// Process stream chunks...
while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  processChunk(value);
}
